# SNHU-Agile-Project
Final project for SNHU course showcasing Agile Scrum methodology, SDLC phases, user stories, and project evaluation through a Sprint Review and Retrospective. 

Sprint Review & Retrospective Reflection
How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?

Interpreting user needs starts with understanding what the user actually wants to accomplish, not just what features they ask for. In this project, I used user stories to break down those needs into simple, clear goals. Writing user stories in the format “as a user, I want to…, so that…” helped me focus on the purpose behind each feature instead of just the technical side. This made it easier to design features that actually solve problems for users, like allowing them to view top destinations or filter trips by preference. Overall, user stories help turn user needs into actionable development tasks.

How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?

I approach developing programs by breaking large tasks into smaller, manageable pieces and working through them step by step. This course showed me how effective Agile processes are, especially using sprints to focus on completing specific goals in short time frames. In the future, I want to continue using Agile practices like maintaining a backlog, prioritizing tasks, and regularly reviewing progress. I also plan to use continuous testing and feedback to improve my work as I go instead of waiting until the end of a project. These processes make development more organized and flexible.

What does it mean to be a good team member in software development?

Being a good team member in software development means communicating clearly, staying accountable, and being willing to collaborate with others. Throughout this project, I learned that strong communication—like asking questions and sharing updates—helps prevent confusion and keeps the team moving forward. It also means being open to feedback and helping others when needed. A good team member focuses not only on their own work but also on the success of the entire team. This mindset leads to better results and a more productive development environment.
